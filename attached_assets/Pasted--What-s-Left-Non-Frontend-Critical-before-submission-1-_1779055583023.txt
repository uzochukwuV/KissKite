## What's Left (Non-Frontend)

**Critical before submission:**

1. **Signal expiry job** тАФ Nothing currently marks signals as `expired` when they pass their deadline without being settled. The scorer settles them, but if `revealSignal` was never called on-chain the signal just rots as `pending` in DB forever. Need a background job that sweeps expired pending signals and marks them `expired` with a reason.

2. **Reveal automation** тАФ The commit-reveal flow is half-automated. Agents commit, but who calls `revealSignal` on-chain before expiration? If that step is manual your autonomy score takes a hit. The scorer or a separate keeper needs to call `revealSignal` before attempting settlement.

3. **Scorer startup validation** тАФ The scorer should check on boot that all required env vars are present (`SCORER_PRIVATE_KEY`, `REPUTATION_REGISTRY_ADDRESS`, `ANTHROPIC_API_KEY`, `API_BASE_URL`, `DATABASE_URL`), that the Kite RPC is reachable, and that the scorer wallet has enough gas. Fail fast with a clear error rather than crashing mid-run during the demo.

4. **Retry logic in scorer** тАФ If the settlement API is down, the scorer should retry with exponential backoff before giving up, not silently drop the settlement.

5. **Demo seed script** тАФ A script that registers demo agents, commits a batch of signals with realistic assets/prices/directions, and fast-forwards some to expired state so the scorer has something to process live during the demo.

6. **README** тАФ Hackathon requirement. Must cover: repo structure, env var list, deploy steps, how to run scorer, how to run API, how to trigger the full loop end-to-end.

7. **Contract verification** тАФ Submit source to Kite explorer if available. Judges will check.

**Nice to have:**
- Docker Compose wiring API + scorer + postgres together
- `GET /api/leaderboard` route ranking agents by on-chain `reputationScore`
